## 状态机定义代码生成工具

### 设计目的

在代码中访问Animator的状态和参数时, 如果直接使用字符串字面量, 会面临以下问题:
- **拼写错误**: 字符串拼写错误在编译时无法发现, 只能在运行时暴露
- **重构困难**: 状态机中的名称修改后, 代码中的字符串不会自动更新
- **可读性差**: 字符串字面量无法提供IDE的自动补全和跳转功能

为了解决这些问题, 我们开发了代码生成工具, 将AnimatorController的结构转换为类型安全的C#定义代码.

### 生成内容

工具会生成一个静态类, 包含三个主要部分:

**1. Layers枚举**:
```csharp
public enum Layers
{
    None = -1,
    Base_0 = 0,
    Base_Show = 1,
    // ...
}
```

**2. Params常量类**:
```csharp
public static class Params
{
    public const string SpeedPerc_XZ = "SpeedPerc_XZ";
    public const string SpeedPerc_Y = "SpeedPerc_Y";
    // ...
}
```

**3. States嵌套类结构**:
按层组织, 每个层包含该层所有状态的常量定义. 对于BlendTree状态, 会生成嵌套的类结构, 包含Self引用和所有子Motion的常量.

### 设计亮点

**标识符合法化**: 工具会自动处理状态机中可能包含的特殊字符、数字开头、C#关键字等情况, 确保生成的代码可以正常编译.

**BlendTree嵌套结构**: 对于复杂的BlendTree结构, 工具会生成扁平化的嵌套类, 避免过深的访问路径. 例如:
```csharp
States.Base_0.BT_Move_Blend.BT_Normal.Ani_Char_Idle
```
这样的结构既保持了层次清晰, 又避免了过长的访问路径.

**唯一性处理**: 如果同一层中存在同名的状态或参数, 工具会自动添加后缀 (`_2`, `_3`等) 确保唯一性.

### 使用方式

在Unity编辑器中, 选中`AnimatorController`资源, 通过右键菜单选择"Assets/动画工具/生成代码". 工具会在指定的输出目录生成`DefAnimatorController_{控制器名称}.cs`文件.

生成的文件包含`<auto-generated>`标记, 明确标识为自动生成, 不应手动编辑. 每次修改状态机后, 重新运行工具即可更新定义代码.

### 实际应用

在项目代码中, 我们可以这样使用生成的定义:

```csharp
// 使用枚举访问层
animator.GetCurrentAnimatorStateInfo((int)DefAnimatorController_Player.Layers.Base_0);

// 使用常量设置参数
animator.SetFloat(DefAnimatorController_Player.Params.SpeedPerc_XZ, speed);

// 使用常量访问状态
var stateInfo = DefAnimatorController_Player.States.Base_0.BT_Move_Blend.BT_Normal.Ani_Char_Idle;
animator.CrossFade(stateInfo.stateName, 0.2f, (int)DefAnimatorController_Player.Layers.Base_0);
```

这样的代码具有以下优势:
- **类型安全**: 编译时检查, 避免拼写错误
- **IDE支持**: 自动补全、跳转定义、重构支持
- **可维护性**: 状态机修改后重新生成代码, 编译错误会提示需要更新的地方
