## 生成Clip-Only状态机工具

### 设计目的

在[丢弃接收端过渡](#丢弃接收端过渡)章节中, 我们提到需要为接收端玩家使用一个"傀儡"状态机, 这个状态机移除了所有需要参数触发的过渡, 只保留可以自动执行的过渡. 手动创建和维护这样的状态机是非常繁琐且容易出错的, 因此我们开发了这个自动化工具.

### 工作原理

工具的核心思路是**结构复制 + 过渡过滤**: 从源状态机复制所有状态和基本属性, 但只保留满足特定条件的过渡.

**状态复制**:
- 递归收集源状态机中的所有状态 (包括子状态机中的状态, 会被"拍平"到同一层级)
- 复制每个状态的基本属性 (motion、speed、mirror等)
- 复制状态上的`StateMachineBehaviour`组件及其序列化参数
- 保持状态机的层级结构和参数定义

**过渡过滤策略**:
工具会保留两类过渡:
1. **无条件Exit过渡**: 状态到Exit的无条件过渡, 用于"一次性"动作的自动退出
2. **ExitTime-only过渡**: 仅依赖`HasExitTime`、无任何条件参数的State到State过渡

这两类过渡的共同特点是**不依赖外部参数**, 可以在接收端自动执行, 不会因为参数不同步而导致状态混乱.

### 使用方式

在Unity编辑器中, 选中需要处理的`AnimatorController`资源, 通过右键菜单选择"Assets/动画工具/生成Clip-Only控制器". 工具会在源控制器同目录下生成一个名为`{原名称}_ClipOnly_DoNotEditDirectly.controller`的新控制器.

**更新机制**: 如果目标文件已存在, 工具会清空其所有层并重新生成, 但保持文件引用不变. 这意味着已经引用该控制器的Prefab或场景不会因为重新生成而丢失引用.

### 技术细节

工具在实现时考虑了以下细节:

- **子状态机拍平**: 子状态机中的状态会被提取到根状态机, 简化结构. 但子状态机上的`StateMachineBehaviour`不会被复制 (会输出警告提示)
- **BlendTree处理**: 如果状态使用BlendTree, 工具会直接引用源BlendTree对象, 不会创建副本
- **默认状态映射**: 尝试将源状态机的默认状态映射到新状态机, 如果映射失败则使用第一个状态作为后备
- **容错处理**: 对于可能在不同Unity版本中不兼容的API, 使用try-catch包裹, 确保工具在不同环境下都能运行

### 实际效果

使用这个工具后, 我们只需要维护一个完整的状态机, 每次修改后重新生成Clip-Only版本即可. 这大大减少了维护成本, 并避免了手动操作可能引入的错误. 更重要的是, 当状态机结构发生变化时 (如新增状态、修改过渡), 只需要重新运行工具即可同步更新, 无需手动调整.
